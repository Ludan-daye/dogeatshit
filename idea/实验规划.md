# 实验规划：用MAUVE验证IQD框架

> 日期：2026年3月

------

## 一、实验总体目标

通过MAUVE分数追踪多代synthetic data训练下的分布偏移，验证以下核心论点：

1. δ来自正则化偏置+有限样本噪声，不是凭空假设
2. 模型在synthetic data上递归训练，IQD逐代衰减，符合 IQD(k) = IQD(0) · αᵏ
3. IQD决定模型性能（而非数据是否"真实"）
4. 存在崩溃阈值 IQD*，低于此值性能急剧恶化
5. 蒸馏不崩而synthetic迭代必崩，原因可用IQD框架统一解释

------

## 二、三层实验设计

### 第零层：可控数学函数环境

**作用**：建立完全透明的基准。真实映射已知，所有误差都可精确归因。最快出结果（CPU，几小时），用来建立直觉、排查bug。

**设置**：
```
真实函数：f*(x) = 2x² + 3x + 1（已知ground truth）
模型：多项式回归 / 小型MLP
数据：从f*加噪采样，ε ~ N(0, σ²)
```

**实验内容**：

#### 实验0a：三组信息损失拆解

| 组别 | 数据类型 | 目的 |
|------|----------|------|
| A | 真实数据（从f*加噪采样） | 基准误差 E₁ |
| B | 完美synthetic（直接从f*重新采样，等价于换一批真实数据） | 误差 E₂ |
| C | 模型生成的synthetic（用A组训练的模型生成预测值） | 误差 E₃ |

**关键量**：
- E₂ - E₁ = 容量/数据量限制（与synthetic无关）
- E₃ - E₂ = **纯粹的生成过程偏差**（即δ的实际影响）
- **作用**：将信息损失精确拆解为两个来源，直接对应δ的理论公式

#### 实验0b：多代迭代

- 在这个环境下跑10-20代：每代用上一代模型的预测值作为下一代训练标签
- 每代记录：与f*的MSE、‖ŵₖ - w*‖（因为w*已知，可以直接算）
- 验证δ累积公式和α的实际值

**预期结果**：
- E₃ - E₂ > 0（生成过程引入偏差）
- ‖ŵₖ - w*‖ 随代数增长，增长模式与理论预测一致

------

### 第一层：Toy Setting（线性回归 + 高斯分布）

**作用**：在理论假设完全匹配的环境下，精确验证Strong Model Collapse的数学公式和IQD衰减。

**设置**：
```
数据：x ~ N(0, Σ),  y = x⊤w* + ε
模型：岭回归 ŵ = (X⊤X + λI)⁻¹X⊤y
迭代：每代用上一代模型的预测值作为下一代的训练标签
```

**实验内容**：

| 编号 | 实验 | 验证目标 |
|------|------|----------|
| 1a | 固定n=1000，变化λ（0.01, 0.1, 1, 10） | δ中正则化偏置分量随λ增大 |
| 1b | 固定λ=0.1，变化n（100, 1000, 10000） | δ中有限样本噪声随n减小 |
| 1c | 跑20代迭代，记录每代的‖δₖ‖和MSE | 验证δ累积公式和α值 |
| 1d | 对1c中每代的输出分布计算MAUVE | 验证MAUVE与‖δₖ‖的单调负相关 |
| 1e | 变化参数量/数据量比例 p/n（0.1 ~ 10） | 验证double descent：插值阈值p/n≈1处崩溃最严重 |

#### 实验1e详细说明（double descent验证）

来自Strong Model Collapse论文的发现：崩溃严重程度随p/n呈非单调变化。

| p/n 区间 | 特征 | 崩溃程度 |
|----------|------|----------|
| p/n << 1 | 欠参数化，学不精确 | 中等（偏差也记不住） |
| p/n ≈ 1  | 插值阈值 | **最严重** |
| p/n >> 1 | 过参数化 | 稍好（信号互相对冲） |

画出崩溃严重程度（ζ / MAUVE下降量）vs p/n 的曲线，验证非单调性。

**预期结果**：
- δ = -λ(X⊤X+λI)⁻¹w* + (X⊤X+λI)⁻¹X⊤ε 精确成立
- ‖δₖ‖随代数增长，增长速率由α决定
- MAUVE与‖δₖ‖强负相关
- p/n ≈ 1 时崩溃项ζ达到峰值

### 第二层：LLM Setting（语言模型 + 文本数据）

**作用**：验证IQD框架在非线性、非高斯的真实场景下依然成立，证明泛化能力。

**设置**：
```
教师模型A：GPT-2 Large（或同级别开源模型），视为"真实世界"
学生模型：GPT-2 Small
数据集：WikiText-103（真实文本基准）
```

**迭代流程**：
```
WikiText-103 → fine-tune 模型₁ → 生成 synthetic₁ (50k条)
           → fine-tune 模型₂ on synthetic₁ → 生成 synthetic₂
           → ...重复至模型₁₀
```

**每一代测量**：

| 指标 | 计算方式 | 作用 |
|------|----------|------|
| MAUVE | mauve.compute_mauve(p_text=真实文本, q_text=模型ₖ输出) | IQD的核心度量 |
| Perplexity | 模型ₖ在真实测试集上的PPL | 辅助验证崩溃 |
| 生成多样性 | distinct-n（n=1,2,3） | 检测模式坍缩 |

**实验内容**：

#### 实验2a：多代纯synthetic迭代
- 10代纯synthetic迭代，画MAUVE衰减曲线
- 拟合 IQD(k) = IQD(0)·αᵏ，报告α和拟合R²

#### 实验2b：混合比例实验
- synthetic占 0%, 25%, 50%, 75%, 100%
- 观察不同比例下单代IQD的下降幅度
- 预期：比例越高，α越小（衰减越快）

#### 实验2c：IQD等价性验证（核心实验）

四组对比设计：

| 组 | 数据来源 | IQD目标 | 控制手段 |
|----|----------|---------|----------|
| A | 真实数据 | ~0.9 | 从WikiText-103采样 |
| B | 高质量synthetic | ~0.9 | 低温度生成（temperature=0.7） |
| C | 加噪真实数据 | ~0.5 | 对真实文本随机替换/打乱 |
| D | 低质量synthetic | ~0.5 | 高温度生成（temperature=1.5） |

- 用**温度参数(temperature)** 控制synthetic质量，使B的MAUVE ≈ A的MAUVE
- 分别用ABCD四组数据训练模型，比较下游任务表现
- **预期**：A ≈ B 且 C ≈ D（IQD决定性能，而非数据是否"真实"）
- **作用**：IQD框架最核心的验证——如果成立，证明"重要的不是真假，而是信息质量"

#### 实验2d：崩溃阈值IQD*
- 逐步降低训练数据的MAUVE（通过调整synthetic比例或温度）
- 找到下游任务性能的断崖位置，即为IQD*
- 给出实践建议："只要 IQD > IQD*，synthetic data 可以安全使用"

#### 实验2e：蒸馏链 vs Synthetic链对比

| 链条 | 流程 | 偏差特征 |
|------|------|----------|
| 蒸馏链 | GPT2-Large → Medium → Small → Tiny | δ来自容量差异（有界） |
| Synthetic链 | Small → synthetic → Small → synthetic → ... | δ来自系统性偏差（会累积） |

- 对比两条链每一步的MAUVE衰减速率
- **预期**：蒸馏链的MAUVE下降缓慢且趋于稳定，synthetic链的MAUVE持续下降
- **作用**：解释现有论文无法回答的问题——"为何蒸馏不崩而synthetic迭代会崩"

## 三、各实验与论文论点的对应关系

| 论文论点 | 验证实验 |
|----------|----------|
| δ来自正则化偏置+有限样本 | 第零层0a（三组拆解）+ 第一层1a/1b |
| δ逐代累积，速率由α决定 | 第零层0b + 第一层1c |
| IQD(k) = IQD(0)·αᵏ | 第一层1d + 第二层2a |
| Synthetic比例越高，衰减越快 | 第二层2b |
| IQD等价性（真假不重要，质量重要） | 第二层2c（四组对比） |
| 崩溃阈值IQD*存在 | 第二层2d |
| 蒸馏不崩 vs synthetic迭代必崩 | 第二层2e |
| 插值阈值处崩溃最严重（double descent） | 第一层1e |

## 四、执行顺序与策略

```
第零层（CPU，几小时）
    │
    ├─ 通过 → 进入第一层
    └─ 不通过 → 调整理论公式或实验设置
                 │
第一层（CPU，1-2天）
    │
    ├─ 通过 → 进入第二层
    └─ 不通过 → 回溯检查理论假设
                 │
第二层（GPU，2-3周）
    │
    └─ 出结果 → 写论文
```

**关键原则**：如果第零层和第一层的结果不符合预期，先调整理论再做第二层，避免浪费GPU资源。

------

## 五、验证方式

- **第零层/第一层**：直接对比实验值与理论公式的数值解，画拟合曲线，报告R²
- **第二层**：用MAUVE衰减曲线拟合指数模型，报告拟合优度；用统计检验（t-test）验证IQD等价性（A≈B、C≈D）
- **所有实验**：设置随机种子，跑3-5次取均值和95%置信区间
